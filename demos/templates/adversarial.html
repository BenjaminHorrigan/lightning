Adversarial Comparison

Strong LLMs already refuse most dangerous requests. The interesting question is not does the LLM refuse? — it's can you trust how it got there? Lightning's value isn't a different decision; it's a provable decision: a named rule that fired, a citation tied to actual CFR, a counterfactual, a deterministic output. Three demonstrations follow.

A · Same Decision, Different Epistemics

3 cases

Both systems refuse. Lightning produces a proof tree, a CFR citation with a real section number, and an explicit counterfactual saying what would change the decision. The LLM produces a paragraph.

B · Hallucinated Citations

5 cases · click to verify

These cases are the cleanest demonstration. Both systems refuse; both produce citations. The LLM's citations are plausible-looking but, when checked against the actual CFR, point to wrong or neighboring sections. Lightning's citations come from citations.json — the source of truth our rules are derived from — so they are verifiable by construction.

C · Determinism

1 artifact · 20 runs each

The same artifact is run 20 times against an LLM at temperature 0.7 and 20 times against Lightning. The LLM's decision flips between REFUSE / ALLOW / ESCALATE; its citations vary in 14 different ways. Lightning produces byte-identical output every time.
