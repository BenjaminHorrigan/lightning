How to Use Lightning

Drop-in safety layer for autonomous research agents. Wrap a single function call around your LLM, agent, or cloud-lab submission and get back a deterministic ALLOW, REFUSE, or ESCALATE with a full proof tree and CFR citations — in under a second.

Quick Start

3 steps · ~60 seconds
  1. 1
    Install
    ▸ shell 
    curl -LsSf https://astral.sh/uv/install.sh | sh
git clone https://github.com/BenjaminHorrigan/lightning
cd lightning
uv sync
  2. 2
    Set your API key
    ▸ .env 
    ANTHROPIC_API_KEY=sk-ant-...
  3. 3
    Run your first check
    ▸ shell 
    uv run lightning check examples/hydrazine_protocol.py
# → REFUSE  (USML IV(h)(1) — controlled propellant)

A · Python SDK

lightning.check()

The single-call entry point. Pass any artifact — a protocol file, a CAD spec, a research proposal — and get a structured ClassificationResult back.

▸ python 
from lightning import check

result = check("examples/hydrazine_protocol.py")

print(result.decision)        # "REFUSE"
print(result.rationale)       # human-readable explanation
print(result.citations)       # [{authority: "ITAR", section: "22 CFR 121.1 IV(h)(1)", ...}]
print(result.proof.steps)     # full ASP proof tree
print(result.confidence)      # 0.0 - 1.0

ClassificationResult schema

▸ pydantic 
class ClassificationResult(BaseModel):
    decision:    Literal["ALLOW", "REFUSE", "ESCALATE"]
    confidence:  float                    # 0.0 - 1.0
    rationale:   str                      # natural-language explanation
    citations:   list[Citation]           # CFR / regulatory references
    proof:       ProofTree                # ASP-derived chain
    artifact:    TechnicalArtifact        # extracted facts (audit trail)
    regimes:     list[str]                # which regimes fired

class Citation(BaseModel):
    authority:   str                      # "ITAR" | "CWC" | "DEA" | ...
    section:     str                      # "22 CFR 121.1 IV(h)(1)"
    text:        str                      # actual regulatory language

class ProofTree(BaseModel):
    steps:       list[ProofStep]
    regime:      str
    rule_file:   str                      # e.g. "usml/cat_iv_propellants.lp"

Three decisions, three flows

ALLOW

No regime fired. Agent proceeds normally with full audit trail of what was checked.

REFUSE

A regime fired with high confidence. Returns rationale + counterfactual ("would be allowed if X were Y").

ESCALATE

Reasoning gap detected. Returns specific question to resolve (parent system, intent, end-user).

B · HTTP API

FastAPI · OpenAPI at /docs

All endpoints accept and return JSON. Run uv run python demos/fastapi_app.py and you also get auto-generated Swagger UI at /docs and ReDoc at /redoc.

POST /api/analyze primary

Run an artifact through the full neurosymbolic pipeline. Extraction (LLM) → reasoning (clingo over 18 regimes) → decision synthesis.

Request
json
{
  "artifact": "Synthesize 50 mg of hydrazine for...",
  "type": "protocol",
  "metadata": { "submitted_by": "agent_42" }
}
Response
json
{
  "decision":   "REFUSE",
  "confidence": 0.97,
  "rationale":  "Hydrazine is a USML IV(h)(1) controlled propellant...",
  "citations": [{
    "authority": "ITAR",
    "section":   "22 CFR 121.1 IV(h)(1)",
    "text":      "Liquid propellants..."
  }],
  "proof":   { "steps": [...], "regime": "usml" },
  "regimes": ["usml", "cwc"]
}

▸ shell
curl -X POST http://localhost:8000/api/analyze \

GET /api/admin/status

Engine health, uptime, memory, recent decision distribution. Powers the Admin console.

POST /api/admin/regimes

Toggle individual regimes on/off without restarting. Useful for benchmarking and reasoning-coverage A/B tests.

POST /api/adversarial/run

Execute the 5-vector adversarial test suite (synonym, circumlocution, split protocol, foreign language, SMILES-only). Returns Lightning vs LLM-baseline detection rates.

GET /api/audit

Cryptographically chained audit log. Each entry includes hash of the previous entry, the artifact, the decision, and a Merkle proof.

WS /ws/agent streaming

Live-intercept channel: an agent streams its planning trace; Lightning injects guard rulings inline as soon as a controlled action is proposed.

C · Agent Wrapper

@lightning_guard

One-line decorator that wraps an agent's tool-call boundary. Works out-of-the-box with ChemCrow; the same pattern fits LangChain Tool, AutoGen functions, and Coscientist task runners.

▸ python · ChemCrow
from chemcrow.agents import ChemCrow
from lightning.integrations.chemcrow import lightning_guard

agent = ChemCrow(model="gpt-4")

@lightning_guard
def run(prompt: str):
    return agent.run(prompt)

# Lightning evaluates the agent's planned actions before execution.
# REFUSE   -> returns refusal + counterfactual instead of running.
# ESCALATE -> returns gap question; await human resolution.
# ALLOW    -> proceeds normally.
result = run("Synthesize 200 mg of fentanyl analog X-1234")
▸ python · LangChain
from langchain.agents import Tool
from lightning.integrations.langchain import LightningGuardedTool

dangerous_tool = Tool(
    name="run_synthesis",
    func=actually_run_in_lab,
    description="Execute a chemical synthesis protocol on the cloud lab.",
)

guarded = LightningGuardedTool.wrap(dangerous_tool)
# guarded behaves like dangerous_tool but never executes a REFUSE'd action.

CLI Reference

uv run lightning check examples/hydrazine_protocol.py

# Pipe stdin
echo "Synthesize 50 mg hydrazine" | uv run lightning check -

uv run lightning check examples/hydrazine_protocol.py --json

# Run only specific regimes
uv run lightning check ARTIFACT --regimes usml,cwc

# Verbose proof output
uv run lightning check ARTIFACT --explain

Common Issues

clingo not found

The reasoning engine needs the clingo binary. uv sync installs the Python wrapper but on some systems you also need:

shell
brew install clingo        # macOS
apt install gringo         # Debian/Ubuntu
ANTHROPIC_API_KEY missing

Required only for the extraction stage. The symbolic reasoning and the golden test suite (tests/test_golden.py) run without any API key; useful for offline benchmarking and CI.

Decision says ESCALATE for everything

Means the extraction layer can't recover one of the slots the reasoning engine needs (parent system, intent, quantity). Open the Admin page and check extraction confidence; raise it or supply explicit metadata in your artifact.

Adding a new regime

Drop a CSV in data/sources/, add the regime to scripts/generate_substance_atoms.py, write a .lp rule file under src/lightning/reasoning/rules/<regime>/, and register it in REGIME_DIRS. See aegis_tier1_claude_code_guide.md for the full pattern.
