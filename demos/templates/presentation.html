02 · A different architecture

Neural + Symbolic.

Neural (LLM)
Reads anything.
◆ Handles synonyms, SMILES strings, foreign language
◆ Extracts substances, quantities, and intent from raw text
◆ Flexible — no fixed input schema required
Weakness: non-deterministic, can be fooled, no proof
+
Symbolic (ASP)
Reasons formally.
◆ Deterministic — same input always yields same answer
◆ Cites the exact regulation and paragraph that fired
◆ Produces a complete, auditable proof tree
Weakness: can't read raw text — needs structured input
=
Lightning
LLM reads and extracts. Symbolic logic decides. Neither can override the other — that separation is the trust property.
Novelty · neural extraction + formal symbolic reasoning — not another guard model
03 · Architecture

Three layers. One trust property.

1 · Neural
Extraction

LLM parses the artifact into a typed TechnicalArtifact — substances, components, quantities, intent.

→
2 · Symbolic
Reasoning

clingo ASP solver runs 514 substances across 18 regimes simultaneously. Produces a ProofTree.

→
3 · Hybrid
Decision

Pure-symbolic decide(); LLM only writes the rationale. ALLOW / REFUSE / ESCALATE with citations.

Trust property

The symbolic layer never calls the LLM. The LLM never overrides the symbolic decision. Extraction and rationale generation are the only LLM roles.
